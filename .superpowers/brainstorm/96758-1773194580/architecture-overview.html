Decisiones clave de diseño

  • Astro como build estático: El dashboard se compila a HTML/JS/CSS y NestJS lo sirve con ServeStaticModule. Elimina la necesidad de un segundo servidor.
  • NestJS como proceso hijo: Electron arranca NestJS como child_process.fork(). Si NestJS falla, Electron puede reiniciarlo sin afectar la UI.
  • License Guard en NestJS: Un interceptor global en NestJS bloquea todas las rutas si la licencia expiró o no fue activada. La UI de activación es la única ruta libre.
  • Machine ID: Se calcula con node-machine-id — usa el UUID del hardware del OS, no la MAC address (que puede cambiar).
