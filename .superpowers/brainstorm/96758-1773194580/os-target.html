¿En qué sistema operativo correrá el software?

Esta decisión afecta el empaquetado, firma de código y distribución del binario

A

Solo Windows

La mayoría de restaurantes usa Windows. Simplifica el desarrollo y el proceso de firma/distribución.

Pro

  • Un solo target
  • Firma .exe estándar
  • Más simple

Contra

  • Limita mercado Mac/Linux
B

Windows + macOS

Cubre el mercado más amplio. Algunos restaurantes premium usan Mac (ej. iPad-like experience con Mac mini).

Pro

  • Más mercado
  • Mac = clientes premium

Contra

  • Doble proceso de firma
  • Apple notarization
  • Más testing
C

Cross-platform (Win + Mac + Linux)

Máxima flexibilidad. Linux útil si el cliente quiere una Raspberry Pi o servidor barato.

Pro

  • Máxima flexibilidad
  • Raspberry Pi / NUC

Contra

  • Más complejidad
  • 3 builds + testing
