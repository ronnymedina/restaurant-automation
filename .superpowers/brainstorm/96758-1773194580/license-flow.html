Flujo de Licencias

Cómo funciona el trial de 15 días, la activación y la validación offline

— Primer arranque —
🚀
1. Electron arranca
El binario se ejecuta. Electron lee el archivo de licencia local antes de arrancar NestJS.
~/.config/restaurant-pos/license.enc
🔍
2. ¿Existe archivo de licencia?
Si no existe → primera instalación. Se crea el archivo con firstLaunchAt y el machineId encriptados con AES-256 usando una clave embebida en el binario.
✓ Existe → leer y validar
✗ No existe → crear trial, arrancar app
⏱
3. ¿Está en período de trial?
Se calcula: hoy - firstLaunchAt. Si el cliente manipula la fecha del sistema, el servidor de licencias lo detectará al activar (las fechas no coincidirán). Para mayor seguridad, también se guarda un timestamp en el registry de Windows / macOS Keychain.
✓ Menos de 15 días → arrancar app normalmente
✗ Más de 15 días → mostrar pantalla de activación
— Activación con licencia —
🔑
4. Cliente ingresa la clave de licencia
Pantalla de activación muestra un campo para la clave (formato: XXXX-XXXX-XXXX-XXXX) y el Machine ID del equipo (para que puedas verificarlo manualmente si hay problemas).
☁️
5. App contacta tu servidor de licencias (única vez)
Envía la clave + machineId a tu API. El servidor verifica que la clave exista, no esté revocada, y no esté registrada en otro machineId diferente.
POST https://licenses.tu-app.com/activate { licenseKey: "XXXX-...", machineId: "abc123...", platform: "win32" }
✓ Válida → devuelve token firmado con tu clave privada
✗ Ya usada en otra PC → error "Licencia en uso"
💾
6. Token firmado se guarda localmente
El token JWT firmado con tu clave privada (RSA-256) se guarda encriptado en disco. Contiene: licenseKey, machineId, activatedAt. En arranques futuros, se verifica con tu clave pública embebida en el binario — sin internet.
{ sub: "XXXX-XXXX", machineId: "abc123", iat: 1741000000 } → firmado con tu RSA private key → verificado con RSA public key (dentro del .exe)
— Arranques subsiguientes (offline) —
✅
7. Verificación offline en cada arranque
Electron lee el token del disco, verifica la firma RSA con la clave pública embebida, y compara el machineId del token con el machineId actual del equipo. Si coinciden → arranca NestJS normalmente. Sin internet requerido.
✓ Firma válida + machineId coincide → app funciona
✗ machineId diferente (copiaron el archivo) → bloqueado
