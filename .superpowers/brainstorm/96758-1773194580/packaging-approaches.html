¿Cómo empaquetar la app como binario de escritorio?

Tres enfoques distintos — cada uno afecta tamaño del binario, experiencia del usuario y complejidad de desarrollo

A

Electron Recomendado

Electron envuelve tu app completa: arranca el servidor NestJS internamente y muestra el dashboard en una ventana nativa.

📦 .exe / .app
Electron Binary
→
🟢 NestJS Server
localhost:3000
→
🖥 Dashboard
ventana Electron
🖥 Kiosk (tablet)
→ LAN →
mismo servidor
·
🍳 Cocina (browser)
→ LAN →
mismo servidor

Pro

  • Firma de código integrada (Win + Mac notarization)
  • Auto-updater con electron-updater
  • Tray icon, inicio automático con Windows
  • Una sola herramienta: Electron Builder
  • Comunidad enorme, bien documentado

Contra

  • Binario grande (~150-200 MB, Chromium incluido)
  • better-sqlite3 requiere recompilación para Electron
  • Mayor uso de RAM

Tamaño: ~150-200 MB Complejidad: Media

B

pkg / Node SEA (binario Node puro)

Empaqueta NestJS + Astro en un único ejecutable. El usuario lo corre y abre su browser en localhost.

📦 .exe / app
pkg binary
→ arranca →
🟢 NestJS Server
localhost:3000
👤 Usuario
→ abre browser →
🌐 localhost:3000

Pro

  • Binario mucho más pequeño (~40-70 MB)
  • Sin Chromium, menos RAM
  • Más fácil de auditar y debuggear

Contra

  • El usuario debe abrir el browser manualmente
  • Menos "app-like" (no hay ventana nativa)
  • better-sqlite3 también necesita prebuilds
  • pkg está en modo mantenimiento (alternativa: @yao-pkg/pkg)
  • Firma de código más manual

Tamaño: ~40-70 MB Complejidad: Alta

C

Tauri + Sidecar

Wrapper en Rust que inicia el servidor NestJS como proceso "sidecar". Usa el webview nativo del OS (no Chromium).

📦 Tauri .exe/.app
Rust shell
→ spawn →
⚙️ NestJS sidecar
proceso separado
→
🖥 WebView nativo

Pro

  • Binario muy pequeño (~10-20 MB)
  • Menor consumo de RAM
  • Firma y notarización integrada

Contra

  • Requiere aprender Rust + Tauri
  • WebView nativo puede variar entre OS
  • Gestión de dos procesos es compleja
  • Curva de aprendizaje alta
  • Socket.IO puede tener issues con webviews

Tamaño: ~10-20 MB Complejidad: Muy Alta
