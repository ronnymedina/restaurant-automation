Desglose de planes de implementación

Cada plan es independiente y ejecutable — ordenados por dependencia

Plan 1 plan-unify-frontend-base Sin dependencias

Migrar restaurant-ui de SSR a static. Sólo infraestructura — sin tocar lógica de negocio.

• Cambiar output: 'server' → output: 'static' y quitar node adapter
• Agregar @astrojs/react (instalado pero sin usarlo aún)
• Verificar que astro build compila sin errores
• Agregar ServeStaticModule en NestJS para servir public/
• Verificar que copy-static.mjs + NestJS sirven el frontend correctamente
🎯 Resultado: pnpm build funciona, NestJS sirve el frontend estático
Plan 2 plan-spa-routing Depende: Plan 1

Configurar NestJS para manejar rutas SPA y renombrar la página del kiosk.

• Wildcard handler en NestJS: /kiosk/* → sirve kiosk/index.html
• Wildcard para /dash/* → sirve dash/index.html
• Renombrar /kiosk/[slug].astro → /kiosk/index.astro
• Leer slug desde window.location.pathname en el script del kiosk existente
🎯 Resultado: todas las URLs del app funcionan en producción sin 404
Plan 3 plan-websocket-to-sse Depende: Plan 1

Reemplazar el WebSocket gateway con Server-Sent Events. Backend y frontend.

• Remover @nestjs/websockets + socket.io del backend
• Crear endpoint @Sse('/v1/events/:restaurantId')
• Emitir evento cuando se crea una orden nueva
• Remover socket.io-client del frontend
• Conectar con EventSource nativo en la página de órdenes
🎯 Resultado: dashboard recibe nuevas órdenes en tiempo real sin WebSocket
Plan 4 plan-react-island-kiosk Depende: Plan 2

Convertir el kiosk (actualmente ~380 líneas de vanilla JS) a componente React.

• Crear src/components/kiosk/KioskApp.tsx
• Migrar toda la lógica del <script> actual a estado React
• Reemplazar manipulación directa del DOM por JSX
• Montar en /kiosk/index.astro con client:load
🎯 Resultado: KioskApp es un componente React con estado reutilizable
Plan 5 plan-react-islands-dashboard Depende: Plan 3

Convertir las páginas del dashboard a React islands. Una por una.

• OrdersLive.tsx — tabla en tiempo real con SSE (usa Plan 3)
• ProductsTable.tsx — CRUD de productos con estado React
• MenusTable.tsx — gestión de menús
• CategoriesTable.tsx — gestión de categorías
• Reemplazar los <script> inline de Astro por client:load
🎯 Resultado: dashboard usa componentes React reutilizables con estado
Plan 6 plan-marketing-homepage Depende: Plan 1

Construir la landing page de marketing en /.

• Diseñar y construir src/pages/index.astro
• Hero, features, CTA hacia /onboarding
• SEO básico (title, meta description, og:tags)
• Responsive — mobile first
🎯 Resultado: landing funcional y SEO-ready servida desde NestJS
