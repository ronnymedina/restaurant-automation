Dos contextos de deployment — mismo código

El sistema debe funcionar tanto en la nube como instalado localmente en la máquina del restaurante

☁️ Cloud / SaaS
Browser
Cliente accede desde cualquier dispositivo
↕
Astro UI (Vercel / Railway / VPS)
Sirve el frontend
↕ HTTP/WS
api-core (NestJS)
REST API + WebSockets + SQLite/Postgres
🖥️ Local / Electron (Desktop)
Ventana Electron
Apunta a http://localhost:PORT
↕
api-core binary (pkg/nexe)
Sirve archivos estáticos + API en el mismo proceso
↕ local DB
SQLite
Archivo local en la máquina del cliente
⚠️ Implicación clave

Para que Electron funcione con un solo proceso, el frontend tiene que ser archivos estáticos que NestJS sirva directamente. Si el frontend es SSR (requiere su propio servidor Node), necesitarías dos procesos en el binary — mucho más complejo.

Esto impacta directamente en la decisión de SSR vs Static SPA.
